# Security Policy

## Reporting a vulnerability

Please **do not** open a public issue for security vulnerabilities.

Report them privately via either:

- GitHub's [private vulnerability reporting](https://github.com/qkniep/pouch/security/advisories/new)
  (Security tab → "Report a vulnerability"), or
- email to <hello@quentinkniep.com>.

Please include a description, reproduction steps, and the affected version. You
can expect an initial response within a few days. Once a fix is available, a
patched release and an advisory will be published, and you will be credited
unless you prefer to remain anonymous.

## Supported versions

Until 1.0, only the latest released version receives security fixes.
